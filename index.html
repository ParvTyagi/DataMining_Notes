Topic 01 · 2 hrs · L1–L3

Introduction to Data Mining

chapter 1.ppt · Data_Mining_Applications.pptx · Han, Kamber Ch.1

📖

What is it?

Data Mining is the process of discovering interesting patterns, correlations, anomalies, and statistically significant structures from large amounts of data stored in databases, data warehouses, or other repositories.

It sits at the intersection of database technology, machine learning, statistics, and pattern recognition. The broader process is called Knowledge Discovery in Databases (KDD) — data mining is just one step in that pipeline: data cleaning → integration → selection → transformation → data mining → pattern evaluation → knowledge presentation.

Think of it this way: you have a mountain of raw transaction logs, medical records, or social media posts. Data mining is the shovel that digs out the gold — useful, non-trivial, previously unknown, and potentially actionable knowledge.

⚡

Key Facts

Data mined
Relational DBs, data warehouses, transactional data, streams, spatial, time-series, text, web, multimedia
Types of patterns
Association rules, classification, regression, clustering, outlier analysis, trend & evolution
KDD steps
Data cleaning → Integration → Selection → Transformation → Mining → Evaluation → Presentation
Interestingness measures
A pattern is interesting if it is valid, novel, useful, and understandable — not all patterns qualify
Major challenges
Scalability, high dimensionality, noisy/incomplete data, privacy & security, streaming data
DM vs ML vs Stats
DM focuses on discovery from stored data; ML on learning models; statistics on inference and hypothesis testing
🎬

Scenarios to Understand It

Retail The Supermarket Mystery
A supermarket chain has 10 years of billing data — millions of receipts. A manager notices sales dip every Tuesday but doesn't know why. A data miner runs pattern discovery and finds: customers who buy diapers on Tuesdays rarely return the same week. Further mining reveals a correlation with paycheck cycles in the neighborhood. This is data mining — a non-obvious, actionable insight hidden in the raw data.
The KDD process here: clean the billing data → integrate with calendar data → mine for patterns → evaluate if the pattern is statistically significant → present as a business report.
Healthcare Hospital Readmission Risk
A hospital has records of 50,000 patients. Without data mining, doctors can only use their experience to predict who will be readmitted within 30 days. With data mining, analysts discover that patients over 65 who are discharged on a Friday with more than 3 medications have a 72% readmission rate. This is a classification pattern — one of the 6 pattern types data mining discovers. The hospital can now flag high-risk patients automatically.
This shows why data mining is different from just querying a database — a SQL query can find "patients readmitted" but cannot discover what conditions predict readmission.
Web / Social Netflix Recommendation
Netflix mines viewing histories of 200 million users. No human could analyze this — data is too large. Data mining finds clusters of users with similar taste, association rules (people who watched Stranger Things also watched Dark), and trends (thriller viewership spikes on rainy weekends). Each of these is a different type of pattern — data mining covers all of them under one umbrella.
Exam Tip
Draw the KDD pipeline from memory and be able to explain what happens at each stage. Short-answer questions frequently ask to "distinguish data mining from data warehousing" or "list the 6 types of data mining tasks with one example each."

Topic 02 · 4 hrs · L4–L7

Data Exploration

Chapter2.ppt · Review of Numericals.pdf · Han, Kamber Ch.2

📖

What is it?

Data Exploration is the preliminary analysis of data to understand its structure, distribution, types, and relationships before applying any mining algorithms. It answers the question: "What does my data look like?"

This topic covers three pillars: (1) Attribute types — understanding what kind of values your columns hold; (2) Statistical descriptions — summarizing the data numerically; (3) Similarity & dissimilarity measures — quantifying how alike two data objects are, which is the foundation of almost every clustering and classification algorithm.

⚡

Key Facts

Attribute types
Nominal (no order, e.g. colour) · Ordinal (ranked, e.g. grades) · Interval (no true zero, e.g. temperature °C) · Ratio (true zero, e.g. salary)
Central tendency
Mean (affected by outliers) · Median (robust) · Mode (most frequent) · Midrange = (max+min)/2
Dispersion
Variance σ², Std dev σ, IQR = Q3−Q1, Range. Box plot shows Q1, median, Q3, whiskers, outliers
Similarity vs distance
Similarity ∈ [0,1]; distance ∈ [0,∞). Higher similarity = more alike. Higher distance = less alike
Minkowski distance
d(i,j) = (Σ|xik−xjk|p)1/p. p=1 → Manhattan; p=2 → Euclidean; p=∞ → Chebyshev
Jaccard coefficient
For binary attributes: J = f₁₁ / (f₀₁ + f₁₀ + f₁₁). Ignores 0-0 matches — useful for sparse data
⚙️

Algorithms / Methods Used

Euclidean Distance
The straight-line distance: d = √(Σ(xᵢ − yᵢ)²). Used when all features are on comparable numeric scales. Default metric in K-Means, KNN. Sensitive to scale differences — always normalize first.
Cosine Similarity
cos(θ) = (A·B) / (|A|·|B|). Measures the angle between two vectors, ignoring magnitude. Ideal for text/document similarity (TF-IDF vectors) where the length of a document shouldn't affect similarity.
Min-Max Normalization
v' = (v − min) / (max − min). Scales all values to [0, 1]. Use before computing any distance metric so that high-range attributes (e.g. salary 0–100K) don't dominate low-range ones (e.g. age 0–100).
Z-score Normalization
v' = (v − μ) / σ. Standardizes to mean 0, std dev 1. Preferred when data has outliers (min-max collapses everything if one value is extreme). Output is not bounded to [0,1].
🎬

Scenarios to Understand It

Statistics Understanding a Student Dataset
Imagine a dataset of 100 students with attributes: Name (nominal), Grade (ordinal — A/B/C/D), Temperature in exam room °C (interval), Marks out of 100 (ratio). You can't average grades (ordinal, no arithmetic). You can average marks (ratio). You can subtract temperatures (interval). Understanding attribute types prevents mathematical errors before you even start mining.
The mean marks may be 72, but if the distribution is skewed (a few very low scorers), median (say 76) is more representative. A box plot instantly reveals this skew — IQR and outlier dots show the full story.
Distance Recommending Similar Movies
Two movies are represented as rating vectors: Movie A = [5, 1, 4] (action, romance, thriller ratings) and Movie B = [4, 1, 3]. Euclidean distance = √((1)²+(0)²+(1)²) = √2 ≈ 1.41. They are close — similar movies. But what if Movie C = [10, 2, 8]? Euclidean distance from A is large, but cosine similarity is high — they point in the same direction. Cosine correctly identifies that C has the same taste profile, just higher ratings overall.
This is why Netflix uses cosine similarity for content vectors, not Euclidean distance — it captures preference direction, not magnitude.
Normalization Why Salary Crushes Age
Customer A: Age=25, Salary=50,000. Customer B: Age=26, Salary=80,000. Euclidean distance without normalization ≈ 30,000 (salary dominates completely; age is invisible). After min-max normalization (age: 0.05, salary: 0.5 for A; age: 0.06, salary: 0.8 for B), distance is now fair. Always normalize before distance-based algorithms.
Exam Tip
Numericals: calculate mean, median, mode, Q1, Q3, IQR from a given dataset and identify outliers using the 1.5×IQR rule. Calculate Euclidean, Manhattan, and Minkowski (p=3) distance between two points. Apply Min-Max normalization to a given value.

Topic 03 · 5 hrs · L8–L12

Data Pre-processing

Chapter3.ppt · Review of Numericals.pdf · Han, Kamber Ch.3

📖

What is it?

Data Pre-processing is the set of transformations applied to raw data before mining to improve data quality and make it suitable for algorithms. Real-world data is almost always incomplete (missing values), noisy (errors, outliers), and inconsistent (conflicting records).

The five major tasks are: Data Cleaning (fix errors), Data Integration (merge sources), Data Reduction (shrink without losing info), Data Transformation (reshape), and Data Discretization (convert continuous to discrete).

"Garbage in, garbage out" — no algorithm can compensate for bad data. Studies estimate 60–80% of a data scientist's time is pre-processing.

⚡

Key Facts

Missing value strategies
Ignore the tuple · Fill with mean/mode · Use most probable value (Bayesian) · Fill manually
Noise handling
Binning (smoothing by bin means/medians/boundaries) · Regression · Clustering to detect outliers
Data reduction
Dimensionality reduction (PCA, wavelet) · Numerosity reduction (sampling, histograms) · Compression
Equal-width binning
Divides range into N equal intervals. Width = (max − min) / N. Simple, but skewed data leaves some bins empty
Equal-frequency binning
Each bin has the same number of values. Better for skewed data. Bins have different widths. Also called equal-depth
Data integration issues
Entity resolution (same real-world entity, different IDs) · Redundancy (correlating attributes) · Data value conflicts
⚙️

Algorithms / Methods Used

Smoothing by Bin Mean
Sort values → divide into bins → replace each value with the bin's mean. Reduces noise. E.g., bin [4, 8, 9] → mean = 7 → all values become 7. Good for smoothing gradual noise.
PCA (Dimensionality Reduction)
Principal Component Analysis projects high-dimensional data onto a lower-dimensional space by finding orthogonal axes (principal components) of maximum variance. Reduces attributes while retaining most information. Combats the curse of dimensionality.
Stratified Sampling
Divide population into strata (subgroups), sample proportionally from each. Ensures rare classes are represented — critical when one class has very few records (e.g., fraud detection where fraud is 0.1% of data).
Decimal Scaling Normalization
v' = v / 10ʲ, where j is the smallest integer such that max(|v'|) < 1. Simple and fast. E.g., values in range [−986, 917]: divide by 1000 → all in [−0.986, 0.917].
🎬

Scenarios to Understand It

Cleaning Hospital Records with Missing Ages
A hospital dataset has 10,000 patient records. 400 patients have missing age values. Strategy options: (a) Delete those 400 rows — loses 4% of data, biased if age-related diseases are over-represented there. (b) Fill with mean age (say 45) — fast but inaccurate for young patients. (c) Fill with most probable value using a regression model on other attributes (gender, diagnosis) — most accurate. The right choice depends on how important age is to the mining task.
For exam: always state which method you'd choose and WHY — marks are given for justification, not just the answer.
Binning Smoothing Salary Data
Raw salaries: 4, 8, 15, 21, 21, 24, 25, 28, 34. Sort them. Equal-width with 3 bins (range=30, width=10): Bin1=[4,8], Bin2=[15,21,21], Bin3=[24,25,28,34]. Smooth by bin mean: Bin1→6, Bin2→19, Bin3→27.75. Now all values in Bin1 become 6. Noise is reduced — the 4 and 8 are treated as the same. Equal-frequency with 3 bins (3 values each): [4,8,15], [21,21,24], [25,28,34].
Equal-frequency better represents the data distribution here — each bin has the same statistical "weight."
Integration Merging Customer Databases
A company merges its online store DB (Customer_ID = "C-1042") with its loyalty card DB (Member_No = "LCM-1042-A"). They refer to the same person but different IDs — entity resolution problem. To detect, correlate on name + email + phone. Also: both DBs have a "purchase_amount" column — one in USD, one in EUR. This is a data value conflict that must be resolved before merging.
Exam Tip
Very common numerical: given a dataset, perform equal-width binning and equal-frequency binning, then smooth by bin mean, bin median, and bin boundaries. Show all intermediate steps. Know the formula for bin width = (max − min) / N.

Topic 04 · 4 hrs · L13–L16

Data Warehouse & OLAP

chapter-4-Data warehouse.ppt · Han, Kamber Ch.4

📖

What is it?

A Data Warehouse is a subject-oriented, integrated, time-variant, and non-volatile collection of data that supports management decision-making. It consolidates data from multiple operational sources into one analytical repository.

OLAP (Online Analytical Processing) provides tools to interactively analyze this multidimensional data. Think of it as a cube where each face represents a different dimension (time, location, product) and each cell holds a measure (sales, revenue). OLAP lets you slice, dice, roll up, and drill down through this cube.

OLTP vs OLAP: OLTP (your bank's transaction system) handles thousands of small, real-time read/write operations. OLAP handles complex analytical queries on historical data — read-heavy, but the queries are expensive.

⚡

Key Facts

4 characteristics of DW
Subject-oriented (organized around topics) · Integrated (uniform format) · Time-variant (historical) · Non-volatile (not deleted/modified)
Star schema
One central fact table surrounded by dimension tables. Simple, denormalized. Fast query performance. Most common DW schema
Snowflake schema
Dimension tables are normalized (split into sub-dimensions). More storage-efficient, but slower queries due to more joins
OLAP operations
Roll-up (aggregate up) · Drill-down (go deeper) · Slice (one dimension fixed) · Dice (multiple dims fixed) · Pivot (rotate)
MOLAP vs ROLAP
MOLAP stores pre-computed cubes (fast queries, large storage). ROLAP queries relational DB on-the-fly (flexible, slower)
AOI (Attribute-Oriented Induction)
Generalizes attribute values by climbing a concept hierarchy (e.g., "Bengaluru" → "Karnataka" → "India"). Produces high-level summaries
🎬

Scenarios to Understand It

Schema Design Retail Sales Data Warehouse
A retail chain wants to analyze sales. Star schema: Fact table = Sales(sale_id, date_id, product_id, store_id, amount). Dimension tables = Date(date_id, day, month, quarter, year), Product(product_id, name, category, brand), Store(store_id, city, state, region). Queries like "Total sales by quarter, by region" are answered with simple star joins — fast. A snowflake variant would split Product into Product + Category + Brand tables — more normalized but slower.
The fact table has foreign keys to all dimension tables. Measures (amounts, quantities) sit in the fact table. Descriptors sit in dimensions.
OLAP Operations The Sales Cube in Action
You're viewing a cube: Dimensions = [Time, Location, Product]. Current view: Monthly sales by city for each product category.

Roll-up on Time: Monthly → Quarterly. You see less granular time data but aggregate sales. Drill-down on Location: City → Neighbourhood. More granular location data. Slice: Fix Time = "Q1 2024". You see a 2D slice (Location × Product) for just Q1. Dice: Fix Time = "Q1–Q2" AND Location = "Bengaluru". A sub-cube. Pivot: Rotate axes so Products become rows and Time becomes columns (like transposing a table).
AOI Generalizing Student Database
You have a student table with city of origin. AOI climbs the concept hierarchy: "Mysuru" → "Karnataka" → "South India" → "India". Instead of a detailed table of 10,000 cities, you get a generalized summary: "45% of students are from South India." This is how data warehouses produce high-level business summaries automatically.
Exam Tip
Drawing a star schema from a given scenario is near-guaranteed. Identify: (1) what are the measures? (2) what are the dimensions? Then draw the fact table in the centre with FK arrows to each dimension. Also memorize the definitions and a one-sentence example of all 5 OLAP operations.

Topic 05 · 12 hrs · L17–L28 · Highest Weightage

Association Rule Mining

Chapter 5.pptx · apriori-algorithm.pdf · fp-growth-algorithm.pdf · pincer_algorithm.docx

📖

What is it?

Association Rule Mining discovers interesting co-occurrence relationships among items in large transaction datasets. The classic example: "Customers who buy bread and butter also tend to buy milk." This rule is written as: bread, butter → milk.

Two key thresholds control what counts as "interesting": Support (how frequently the itemset occurs overall) and Confidence (how often the rule holds when the left side is true). A rule is accepted only if both exceed user-defined minimums (minsup and minconf).

This topic covers four algorithms that find frequent itemsets with different strategies — each trading off speed, memory, and database scans.

⚡

Key Facts

Support
support(A→B) = P(A∪B) = transactions containing both A and B / total transactions. Measures how often the itemset appears
Confidence
confidence(A→B) = P(B|A) = support(A∪B) / support(A). Measures rule reliability when A is present
Anti-monotone property
All subsets of a frequent itemset must be frequent. Equivalently: if an itemset is infrequent, all its supersets are infrequent. This is Apriori's core pruning principle
Lift
lift(A→B) = confidence(A→B) / support(B). Lift > 1 means A and B are positively correlated. Lift = 1 means independent. Used to filter trivial rules
Closed frequent itemset
An itemset X is closed if no proper superset of X has the same support. Mining closed sets reduces output without losing information
ECLAT vertical format
Instead of storing transactions, store the set of transaction IDs (tidset) per item. support(A∪B) = |tidset(A) ∩ tidset(B)|. No database scans after initial load
⚙️

Algorithms Used

Apriori Algorithm
Generate-and-test approach: (1) Scan DB, find frequent 1-itemsets (L₁). (2) Join Lₖ with itself to generate Cₖ₊₁ candidates. (3) Prune candidates using anti-monotone property — any candidate with an infrequent subset is dropped. (4) Scan DB to count support of remaining candidates. Repeat. Weakness: multiple DB scans (one per itemset size k); huge candidate sets. Complexity: O(2ⁿ) candidate sets worst case.
FP-Growth Algorithm
Divide-and-conquer, no candidate generation: (1) Scan DB once — build a compact FP-tree (Frequent Pattern tree). Each path = a transaction, nodes share prefixes. (2) For each frequent item, extract its conditional pattern base (prefix paths). (3) Build a smaller conditional FP-tree. (4) Mine patterns recursively. Only 2 DB scans total. 10–100× faster than Apriori on dense data.
Pincer Search Algorithm
Bi-directional search: Apriori searches bottom-up (from 1-itemsets up). Pincer simultaneously searches top-down from a Maximal Frequent Itemset Boundary (MFIB). If an itemset is a superset of an MFIB member, it is infrequent — pruned from top. This drastically reduces the number of candidates generated, especially when maximal frequent itemsets are large.
ECLAT (Vertical Format)
Tidset intersection approach: Converts horizontal format (transaction → items) to vertical format (item → tidset). Support of any itemset = |intersection of tidsets|. No scanning needed after initial conversion — just set intersections. Extremely fast for sparse datasets. Memory-intensive for very large tidsets.
🎬

Scenarios to Understand It

Apriori Market Basket Analysis Step-by-Step
5 transactions, minsup = 60% (≥ 3 transactions): T1={A,B,C}, T2={A,C}, T3={A,B}, T4={A,B,C,D}, T5={B,C}.

L₁: A(4), B(4), C(4), D(1) → D dropped (support < 3). Frequent: {A},{B},{C}.
C₂: {A,B}(3✓), {A,C}(3✓), {B,C}(3✓). All frequent → L₂ = all.
C₃: {A,B,C} — check subsets: {A,B}✓{A,C}✓{B,C}✓ → not pruned. Count: T1,T4 → support=2 → dropped.
Final frequent itemsets: {A},{B},{C},{A,B},{A,C},{B,C}.
Rule from {A,C}: A→C: conf = 3/4 = 75%. C→A: conf = 3/4 = 75%.
Key lesson: the anti-monotone property prunes {A,B,C} because its support (2) is below minsup (3). We didn't even need to count — the rule killed it before the scan.
FP-Growth Building the FP-Tree
Same dataset. Step 1: Find frequent items in order of frequency: A:4, B:4, C:4. Step 2: Reorder each transaction by frequency — T1: A,B,C; T2: A,C; T3: A,B; T4: A,B,C; T5: B,C. Step 3: Insert into tree: each item is a node; shared prefixes share paths. The tree compresses 5 transactions into ~7 nodes. Step 4: For item C (least frequent), backtrack all paths containing C to get conditional pattern base = {A,B:2, A:1, B:1}. Build conditional FP-tree for C, mine recursively.
FP-Growth never generated {A,B,C,D} as a candidate at all — it only explores what's in the compressed tree. No wasteful candidate generation.
Real World Amazon's "Frequently Bought Together"
Amazon mines billions of orders using association rule mining. The rule {Laptop charger, USB hub} → {Laptop stand} with confidence 68% and lift 3.2 means: customers who buy a charger and hub are 3.2× more likely to also buy a stand than a random customer. Amazon uses this to power the "Frequently Bought Together" section. Because the dataset is huge (millions of items), Apriori is impractical — FP-Growth or parallel variants are used instead.
ECLAT Vertical Format Support Counting
Convert to vertical: A→{T1,T2,T3,T4}, B→{T1,T3,T4,T5}, C→{T1,T2,T4,T5}. Support of {A,B} = |{T1,T3,T4} ∩ {T1,T3,T4,T5}| = |{T1,T3,T4}| = 3 ✓. No database scan needed — just list intersections. For large, sparse datasets this is extremely fast because tidsets are small.
Exam Tip
This is your highest-marks topic. Be able to: (1) Run full Apriori iteration on a given transaction table, showing C₁→L₁→C₂→L₂ etc. (2) Build an FP-tree from scratch and extract the conditional pattern base for a given item. (3) Calculate support, confidence, and lift for given rules. (4) Explain why Pincer is faster than Apriori and why FP-Growth avoids candidate generation.

Topic 06 · 5 hrs · L29–L33

Classification

Chapter 6.pptx · Decision tree.pptx · 1-3 algorithm docx files · numericals solution PDF

📖

What is it?

Classification is a supervised learning task where a model is trained on labelled examples and then predicts the class label for new, unseen instances. It is supervised because training data includes the correct answers (labels).

The process has two phases: Training (build a model from labelled data) and Testing/Prediction (use the model to classify new data). The goal is to learn a mapping from attribute values → class label that generalises to unseen data.

This topic covers four approaches: Decision Trees (rule-based splits), Naive Bayes (probabilistic), Rule-Based Classification (explicit IF-THEN rules), and Ensemble Methods (combining multiple models for higher accuracy).

⚡

Key Facts

Information Gain (ID3)
IG(A) = Entropy(S) − Σ (|Sᵥ|/|S|) · Entropy(Sᵥ). Select attribute with highest IG. Biased toward many-valued attributes
Entropy
H(S) = −Σ pᵢ log₂(pᵢ). Measures impurity. H=0 → pure node (all one class). H=1 → perfectly mixed (50/50 binary)
Gini Index (CART)
Gini(S) = 1 − Σ pᵢ². Gini=0 → pure. Gini=0.5 → 50/50. Split on attribute that gives lowest weighted Gini
Naïve Bayes
P(C|X) ∝ P(C) · Π P(xᵢ|C). Assumes conditional independence of attributes given class. Fast and works well with small data
Laplace Smoothing
P(xᵢ|C) = (count + 1) / (total + k). Prevents zero probabilities when an attribute value never appears with a class in training data
Model evaluation
Accuracy, Precision = TP/(TP+FP), Recall = TP/(TP+FN), F1 = 2·P·R/(P+R). Use confusion matrix. Cross-validation for small datasets
⚙️

Algorithms Used

ID3 / C4.5 Decision Tree
Top-down greedy induction: at each node, select the attribute with maximum Information Gain (ID3) or Gain Ratio (C4.5 — fixes bias toward many-valued attrs). Split the dataset. Recurse. Stop when a node is pure or no attributes remain. C4.5 adds post-pruning to avoid overfitting. Tree is human-readable and handles both categorical and numeric attributes.
CART (Gini-based)
Always binary splits: for each attribute and each possible split point, compute weighted Gini impurity of the two child nodes. Choose the split that minimizes Gini. Repeat. CART produces a binary tree — every internal node has exactly two children. Better for numeric attributes than ID3.
Naïve Bayes Classifier
For each class C, compute P(C) × Π P(xᵢ|C). Assign the class with the highest product. Uses the conditional independence assumption — attributes don't influence each other given the class. Empirically works very well despite this naive assumption (attributes are usually correlated in reality). Apply Laplace smoothing when counts = 0.
Bagging (Random Forest)
Bootstrap aggregating: train T different decision trees, each on a random subset (with replacement) of training data. Final prediction = majority vote. Reduces variance (overfitting). Random Forest additionally selects a random subset of features at each split — prevents trees from being correlated.
Boosting (AdaBoost)
Sequential ensemble: train a weak classifier. Up-weight misclassified samples. Train next classifier on reweighted data. Combine all classifiers with weighted voting. Reduces bias. More accurate than Bagging in many cases but more prone to overfitting on noisy data.
🎬

Scenarios to Understand It

Decision Tree Loan Approval System
A bank has 1000 historical loan applications labelled "Approved" or "Rejected." Attributes: Age (young/middle/old), Income (low/medium/high), Credit History (bad/good). Build a decision tree: first compute Information Gain for each attribute. Suppose Income has IG=0.3, Age has IG=0.15, Credit has IG=0.4. Credit is selected as root. Branch on bad/good. In the "good" branch, Income is the next best splitter. The tree learns: "if Credit=good AND Income=high → Approve."
This tree is transparent — you can explain the decision to the loan applicant. Neural networks cannot do this. That's why decision trees are used in regulated industries (banking, healthcare).
Naïve Bayes Email Spam Detection
Training data: 100 spam emails, 200 ham emails. In spam, the word "free" appears in 70% of emails: P(free|spam)=0.7. In ham, P(free|ham)=0.05. P(spam)=100/300≈0.33. For a new email containing "free": P(spam|free) ∝ 0.33 × 0.7 = 0.231. P(ham|free) ∝ 0.67 × 0.05 = 0.0335. 0.231 > 0.0335 → classified as Spam. If "win" also appears and P(win|spam)=0.6, P(win|ham)=0.01: P(spam|free,win) ∝ 0.33 × 0.7 × 0.6 = 0.1386. P(ham|free,win) ∝ 0.67 × 0.05 × 0.01 = 0.000335. Overwhelmingly spam.
The independence assumption means we just multiply — if words were correlated, this would give wrong probabilities, but in practice spam filters work extremely well with Naïve Bayes.
Gini Index When to Use CART over ID3
You have a continuous attribute "Age" with values 22, 25, 28, 35, 50, 60. ID3 can't directly handle continuous attributes — you'd need to discretize first. CART evaluates every possible split point (e.g., Age ≤ 28 vs Age > 28) and computes Gini for each. This makes CART more natural for numeric data. Also: CART always produces binary trees — simpler to interpret and prune than multi-branch ID3 trees.
Ensemble Why One Tree Isn't Enough
Imagine predicting whether a patient has a rare disease. A single decision tree trained on 500 samples overfits — it memorizes training data and fails on new patients (high variance). Random Forest trains 100 trees, each on a random 500-sample bootstrap of the data, and votes. The error of 100 imperfect classifiers combined is far lower than any single perfect-seeming classifier. This is why Random Forest is one of the most widely-used algorithms in practice.
Exam Tip
Mandatory practise: (1) Calculate entropy and information gain for a weather/play dataset. (2) Calculate Gini index for a 2-class split. (3) Compute Naïve Bayes posterior probabilities with Laplace smoothing. (4) Fill in a confusion matrix and derive accuracy, precision, recall, F1. Your numericals solution PDF has all these worked out — study those examples.

Topic 07 · 4 hrs · L34–L37

Clustering

Clustering basics.ppt · K means solved example.docx · Agglomeritive.docx · DIANA & DB scan.docx

📖

What is it?

Clustering is an unsupervised learning task — data has no labels. The goal is to group data objects so that objects within the same cluster are similar to each other (high intra-cluster similarity) and objects in different clusters are dissimilar (low inter-cluster similarity).

Unlike classification, there is no "correct answer" to learn from. The algorithm must discover natural groupings on its own. This is useful when you don't know the categories in advance — e.g., discovering customer segments, gene families, or document topics.

Four families of methods: Partitioning (K-Means, K-Medoids), Hierarchical (Agglomerative, DIANA), Density-based (DBSCAN), and Grid-based.

⚡

Key Facts

K-Means steps
1. Pick k centroids randomly. 2. Assign each point to nearest centroid. 3. Recompute centroids as means. 4. Repeat until no change
K-Means limitations
Sensitive to outliers (centroid is pulled). Requires specifying k in advance. Assumes spherical clusters. Result varies with initial centroid choice
K-Medoids (PAM)
Centroid is replaced by an actual data point (medoid). Robust to outliers. More computationally expensive: O(k(n−k)²) per iteration
Hierarchical linkage
Single (min distance, chain-like), Complete (max distance, compact), Average (balanced), Ward (minimizes variance increase)
DBSCAN parameters
ε (epsilon): neighbourhood radius. MinPts: min points to be a core point. Core, border, and noise points are defined relative to these
Cluster evaluation
SSE (Within-cluster sum of squares — lower is better). Silhouette coefficient ∈ [−1,1] — close to 1 = well-clustered; near 0 = border; negative = wrong cluster
⚙️

Algorithms Used

K-Means
Objective: minimise SSE = Σᵢ Σₓ∈Cᵢ ||x − μᵢ||². Iterative coordinate descent on cluster assignments and centroids. Complexity: O(nkt) where n=points, k=clusters, t=iterations. Fast and simple. Fails on non-spherical clusters and when outliers distort centroids.
K-Medoids (PAM)
Build phase: initialise medoids. Swap phase: for each (medoid, non-medoid) pair, compute cost change if they swap. If swapping reduces total cost (sum of distances to nearest medoid), swap. Repeat until no improvement. The use of actual data points as centres makes this robust when outliers exist.
Agglomerative HAC
Bottom-up: start with N clusters (one per point). Compute all pairwise distances. Merge the two closest clusters. Update distance matrix. Repeat until 1 cluster. The merge sequence produces a dendrogram — cut it at any height to get any number of clusters. Linkage criterion determines cluster shape (single → elongated; complete → compact).
DIANA (Divisive)
Top-down: start with 1 cluster containing all points. Find the point with the highest average dissimilarity to all others in the cluster — this becomes the "splinter group" seed. Reassign points that are closer to the splinter group than to the main cluster. Continue until all clusters are singletons. More suitable when you know the number of clusters at the top level.
DBSCAN
A point p is a core point if ≥ MinPts points lie within ε. A border point is within ε of a core point but has fewer than MinPts neighbours. A noise point is neither. Clusters grow by connecting density-reachable core points. Advantages: finds arbitrary shapes, automatically identifies outliers as noise, no need to specify k. Weakness: struggles with varying density clusters.
🎬

Scenarios to Understand It

K-Means Customer Segmentation
An e-commerce company has 10,000 customers with purchase frequency and average spend. They run K-Means (k=3). Initial centroids: (5, 200), (20, 800), (50, 5000). Iteration 1: assign each customer to nearest centroid. Recompute: centroid 1 becomes (6, 210), centroid 2 becomes (22, 750), centroid 3 becomes (48, 4800). Iteration 2: reassign based on new centroids — some customers switch clusters. Repeat until stable. Result: Cluster 1 = occasional low spenders, Cluster 2 = regular mid spenders, Cluster 3 = high-value customers. Marketing team can now target each segment differently.
This is unsupervised — nobody told the algorithm what "segment" means. It discovered the natural groupings from data alone.
Hierarchical Building a Dendrogram from Cities
5 cities with pairwise distances: A-B: 2, A-C: 8, B-C: 7, A-D: 10, B-D: 9, C-D: 4, all-E: large. Using single linkage (merge closest pair): Step 1 → merge A,B (dist=2) → cluster {A,B}. Step 2 → distance from {A,B} to C = min(7,8)=7; {A,B} to D = min(9,10)=9; C to D = 4 → merge C,D (dist=4). Step 3 → {A,B} to {C,D} = 7 → merge. The dendrogram shows exactly when each merge happened. Cut at height 5 → 2 clusters: {A,B} and {C,D,E...}.
The power of hierarchical clustering: one run gives you the dendrogram, and you choose your desired number of clusters by cutting at any height. K-Means requires re-running for each k.
DBSCAN Finding Earthquake Epicentre Clusters
Seismologists have 50,000 GPS coordinates of earthquake epicentres. Some areas have dense clusters (fault lines), others have isolated events. K-Means would force spherical clusters and incorrectly group events near curved fault lines. DBSCAN with ε=10km and MinPts=5 correctly finds: long, curved high-density clusters (fault lines), sparse background noise (random isolated events classified as noise points). The noise points are themselves interesting — they may indicate previously unknown fault activity.

(2) Agglomerative clustering — start from a given distance matrix, show merges step by step, and draw the dendrogram. Topic 08 · 4 hrs · L38–L41

Outlier Detection

Outlier introduction.ppt · outlier numericals.docx · Univariate outlier calculation.docx · grubb's values.jpg

📖

What is it?

An outlier is a data object that deviates significantly from the rest of the dataset — so much so that it appears to have been generated by a different mechanism. Outliers are sometimes called anomalies, exceptions, or surprises.

Critically, outliers are not always errors. A fraudulent credit card transaction is an outlier AND the most interesting thing in the data. Outlier detection is therefore a primary task in fraud detection, network intrusion, medical diagnosis, and industrial fault detection.

Four main approaches: Statistical (model data distribution), Proximity-based (distance/density from neighbours), Clustering-based (far from any cluster), Classification-based (train on normal data only).

⚡

Key Facts

Global vs local outliers
Global: outlier w.r.t. entire dataset. Local (contextual): outlier only in a local neighbourhood — e.g., 25°C is normal globally but an outlier if it's winter in Delhi
IQR method
Outlier if value < Q1 − 1.5×IQR or > Q3 + 1.5×IQR. Non-parametric — no distribution assumption. Robust and simple
Grubbs' test statistic
G = max|xᵢ − x̄| / s. Compare G to critical value from Grubbs table (given α, n). If G > G_critical → outlier detected
LOF (Local Outlier Factor)
LOF(p) = avg(lrd(o)) / lrd(p) for o in neighbours. LOF >> 1 → p is in a much sparser region than its neighbours → outlier
Clustering-based
DBSCAN noise points are outliers by definition. In K-Means, points far from any centroid (high distance to nearest centroid) are outliers
Types of outliers
Point outliers (single data point), Contextual outliers (outlier in context), Collective outliers (a group of points is unusual even if individual points are normal)
⚙️

Algorithms Used

Grubbs' Test (Statistical)
Assumes data is normally distributed. Computes G = |x_suspect − x̄| / s. Compare against critical value G_α,n from the Grubbs table. If G exceeds the critical value, the suspect value is statistically an outlier at significance level α. Univariate only. Your grubb's values.jpg contains the critical value table — memorise how to read it (row = n, column = significance level).
IQR Method
Sort data → compute Q1, Q3 → IQR = Q3 − Q1. Lower fence = Q1 − 1.5×IQR. Upper fence = Q3 + 1.5×IQR. Any point outside fences = outlier. More robust than Grubbs because it makes no distributional assumption — even works on highly skewed data. This is what a box plot's whiskers represent.
LOF Algorithm
1. For each point p, find its k nearest neighbours. 2. Compute reachability distance from p to each neighbour o. 3. Compute local reachability density (lrd) of p = 1 / (avg reachability distance). 4. LOF(p) = avg(lrd of p's neighbours) / lrd(p). LOF ≈ 1 → normal. LOF >> 1 → outlier. Key advantage: detects local outliers in datasets with varying density — unlike global distance methods.
🎬

Scenarios to Understand It

Statistical Detecting a Fraudulent Transaction
A credit card user's transaction history: daily spend in rupees over 30 days with mean ₹1200 and std dev ₹300. One day they spend ₹15,000. Grubbs: G = |15000 − 1200| / 300 = 46. G_critical at α=0.05, n=30 is about 3.1. 46 >> 3.1 → statistically significant outlier. Bank flags the transaction for review. This is a point outlier and a global outlier (unusual in the entire history).
The transaction might still be legitimate (user bought a laptop). Outlier detection flags for review — a human or further system makes the final call.
IQR Method Quality Control in Manufacturing
Bottle fill levels (ml): 498, 500, 499, 501, 498, 500, 499, 502, 498, 550, 499. Q1=498.5, Q3=500.5, IQR=2. Lower fence=498.5−3=495.5, Upper fence=500.5+3=503.5. The value 550 ml exceeds 503.5 → flagged as outlier. The bottle was overfilled — a sensor malfunction or faulty nozzle. This is caught by IQR without needing to know the fill distribution.
LOF Network Intrusion Detection
A server logs network packets. Most legitimate traffic forms dense clusters (normal browsing patterns). An attacker sends packets at unusual intervals with unusual port combinations. These are a collective outlier — individually each packet might look normal, but the pattern of the group is unusual. LOF detects this because the attacker's packets have a very low local reachability density compared to normal traffic clusters. This is why LOF is preferred over Grubbs in network security — the data has multiple varying-density regions.
Contextual Weather Anomaly
Temperature 38°C in Chennai in May: not an outlier — typical for the season. Temperature 38°C in Shimla in May: contextual outlier — highly unusual for that location and month. Simple statistical methods on raw temperature values would miss this. Contextual outlier detection requires modelling the context (location, season) and detecting outliers within that context — similar to LOF but the "neighbourhood" is defined by context attributes, not just spatial proximity.
Exam Tip
Practise Grubbs' test end-to-end: compute mean and std dev of dataset → find the most extreme value → compute G → look up critical value from grubb's values.jpg → state conclusion. Also practise IQR-based outlier detection from Univariate outlier calculation.docx. Know all three outlier types (point, contextual, collective) with one example each.

Data Mining Applications

Web Mining.pptx · Data_Mining_Applications.pptx · Recommender_Systems_Case_Study_QnA.pdf · Temporal_Spatial.pdf

📖

What is it?

This unit applies data mining techniques to specific real-world domains. Three major application areas are covered: Recommender Systems (personalising suggestions), Web Mining (mining web content, structure, and usage), and Temporal & Spatial Data Mining (patterns that depend on time and geography).

Each domain introduces unique challenges: web data is semi-structured and hyperlinked; temporal data has ordering and seasonality; spatial data has geographic neighbourhood relationships. General data mining algorithms often need modification to handle these characteristics.

⚡

Key Facts

Collaborative filtering
User-based CF: find similar users → recommend what they liked. Item-based CF: find similar items → recommend similar to what user liked. Uses cosine/Pearson similarity
Content-based filtering
Recommends items similar to what the user has liked before, based on item attributes (genre, author, tags). Uses TF-IDF to represent content. Doesn't need other users
Cold start problem
CF fails for new users (no history) or new items (no ratings). Content-based partially solves for new items. Hybrid approaches combine both
Web mining types
Content mining (text, images on pages), Structure mining (hyperlinks between pages — PageRank), Usage mining (server logs — clickstreams)
PageRank
PR(p) = (1−d)/N + d × Σ PR(q)/L(q). A page's rank = weighted sum of ranks of pages linking to it. Iterative computation until convergence. d=damping factor (0.85)
Temporal patterns
Sequential patterns: Customer buys A, then B, then C over time. Time-series clustering: group similar time series by shape. Trend analysis: detect rising/falling patterns
⚙️

Algorithms Used

User-based Collaborative Filtering
1. Build user-item rating matrix. 2. Compute pairwise user similarity (Pearson correlation or cosine). 3. For target user, find k most similar users (neighbours). 4. Predict rating = weighted average of neighbours' ratings. 5. Recommend items with highest predicted rating. Weakness: doesn't scale to millions of users — pairwise computation is O(n²).
Item-based CF (Amazon's approach)
Compute item-item similarity (cosine of rating vectors). For a user's rated items, find similar items they haven't rated. Recommend those. Item similarities are stable over time — precompute and cache. More scalable than user-based CF for large item catalogues because number of items < number of users (usually).
PageRank
Models the web as a directed graph (pages = nodes, hyperlinks = edges). PageRank = probability that a random surfer lands on a page after infinitely many random clicks. Iteratively update PR(p) until convergence. Pages with many high-quality inbound links rank higher. The damping factor d (usually 0.85) models the probability of the surfer continuing (vs jumping to a random page).
TF-IDF (Content-based)
TF(t,d) = (count of term t in document d) / (total terms in d). IDF(t) = log(N / df(t)), where df = documents containing t. TF-IDF = TF × IDF. Gives high weight to terms that are frequent in a document but rare across all documents — these are the most distinctive terms. Used to represent document content as vectors for similarity computation.
🎬

Scenarios to Understand It

Recommender Spotify's Music Recommendations
User A and User B both liked artists X, Y, Z. User B also likes artist W. User A hasn't heard W. User-based CF: A and B are similar users → recommend W to A. Now consider: Spotify has 600 million users. Computing all pairwise similarities = 1.8×10¹⁷ operations — impossible. So Spotify uses item-based CF: precompute song-song similarity. When A plays song P, find songs most similar to P and recommend those. This runs in milliseconds because the similarity matrix is precomputed.
Spotify also uses audio features (tempo, key, energy) for content-based filtering — making it a hybrid recommender system.
Web Mining How Google Ranks Pages
Web structure mining: Google treats the web as a graph. A link from page A to page B is a "vote" for B. A link from a high-PageRank page (say Wikipedia) is worth more than one from a personal blog. PageRank iterates: PR(A) = 0.15/N + 0.85 × (PR(Wikipedia)/L_wiki + ...). After ~50 iterations, PageRank values converge. Pages with high PageRank appear first in search results. This is why SEO focuses on getting backlinks from authoritative sites.
Temporal Mining E-commerce Purchase Sequences
Sequential pattern mining on customer purchase history: A customer first buys a camera, then a memory card, then a camera bag — across three separate transactions over two weeks. This is a sequential pattern: camera → memory card → bag. Finding this pattern across thousands of customers allows e-commerce sites to send targeted emails: "You bought a camera 3 days ago — here are memory cards." This uses temporal ordering, not just co-occurrence (association rules).
Spatial Mining Crime Hotspot Analysis
A city police department has GPS coordinates of 50,000 reported crimes over 5 years. Spatial clustering (DBSCAN on coordinates) reveals crime hotspots — dense clusters of incidents. These hotspots are spatially correlated (nearby regions have similar crime rates). Spatial association rules: "Regions near bars with poor lighting have 3× higher crime rate." This is a spatial association rule where the relationship is geographic proximity, not transactional co-occurrence.
